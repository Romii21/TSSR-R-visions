1. **Un conteneur est éphémère** : quand tu le supprimes, tout disparaît (sauf si tu utilises des volumes)
    
2. **Les images sont stockées localement** : après un `docker pull`, l'image reste sur ton disque
    
3. **Docker Hub** : c'est comme un "app store" d'images Docker (hub.docker.com)
    
4. **Isolation** : chaque conteneur ne voit que ce qui est à l'intérieur de lui-même
    
5. **Root par défaut** : dans un conteneur, tu es souvent utilisateur `root`
    

## 🚀 Workflow typique

```bash
1. docker pull ubuntu           # Télécharger l'image
2. docker run -it ubuntu bash   # Créer et lancer le conteneur
3. # Faire ton travail dans le conteneur
4. exit                         # Sortir du conteneur
5. docker ps -a                 # Voir tous les conteneurs
6. docker rm <id>               # Nettoyer
```

## 💡 Astuces

- Utilise `--name` pour donner des noms faciles à retenir à tes conteneurs
- `docker ps -q` affiche seulement les IDs (pratique pour les scripts)
- Tu peux utiliser les premiers caractères d'un ID au lieu de l'ID complet
- `Ctrl+P` puis `Ctrl+Q` pour sortir d'un conteneur sans l'arrêter

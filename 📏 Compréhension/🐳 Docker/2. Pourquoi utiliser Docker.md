✅ **Portabilité** : Ça marche pareil sur n'importe quelle machine (Windows, Mac, Linux)  
✅ **Isolation** : Chaque application vit dans son propre environnement  
✅ **Simplicité** : Plus besoin d'installer plein de dépendances manuellement  
✅ **Reproductibilité** : L'application fonctionne exactement pareil partout

## 🧩 Les concepts clés

### Image

Une **image** = le modèle, la recette de ton application

- C'est un paquet qui contient tout (code + dépendances)
- Elle est **immuable** (elle ne change pas)
- On peut la télécharger depuis Docker Hub

### Conteneur

Un **conteneur** = une instance en cours d'exécution d'une image

- C'est comme un mini-ordinateur isolé qui tourne sur ta machine
- Il a son propre système de fichiers, ses processus, son réseau
- Quand tu arrêtes le conteneur, il disparaît (sauf si tu sauvegardes)

### Analogie

- **Image** = recette de gâteau
- **Conteneur** = le gâteau que tu as cuisiné avec cette recette

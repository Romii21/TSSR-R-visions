### Gestion des Images

```bash
# Télécharger une image
docker pull <nom-image>
docker pull ubuntu
docker pull nginx:latest

# Lister les images disponibles
docker images
docker image ls

# Supprimer une image
docker rmi <nom-image>
docker image rm <nom-image>
```

### Gestion des Conteneurs

```bash
# Lancer un conteneur
docker run <nom-image>
docker run hello-world

# Lancer en mode interactif (avec terminal)
docker run -it <nom-image> bash
docker run -it ubuntu bash

# Lancer en arrière-plan (daemon)
docker run -d <nom-image>
docker run -d nginx

# Lancer avec un port exposé
docker run -p <port-hôte>:<port-conteneur> <nom-image>
docker run -p 8080:80 nginx

# Lister les conteneurs actifs
docker ps

# Lister TOUS les conteneurs (même arrêtés)
docker ps -a

# Arrêter un conteneur
docker stop <id-conteneur>

# Démarrer un conteneur arrêté
docker start <id-conteneur>

# Supprimer un conteneur
docker rm <id-conteneur>

# Voir les logs d'un conteneur
docker logs <id-conteneur>
```

### Options utiles pour `docker run`

```bash
-d              # Détaché (en arrière-plan)
-it             # Interactif + terminal
-p 8080:80      # Port mapping (hôte:conteneur)
-e VAR=value    # Variable d'environnement
-v /host:/cont  # Volume (partage de fichiers)
-h hostname     # Nom d'hôte du conteneur
--name monapp   # Donner un nom au conteneur
--rm            # Supprimer automatiquement après arrêt
```

### Commandes de nettoyage

```bash
# Nettoyer les ressources non utilisées
docker system prune

# Supprimer tous les conteneurs arrêtés
docker container prune

# Supprimer toutes les images non utilisées
docker image prune
```

### Commandes avancées

```bash
# Exécuter une commande dans un conteneur actif
docker exec -it <id-conteneur> bash

# Copier des fichiers hôte → conteneur
docker cp fichier.txt <id-conteneur>:/chemin/

# Inspecter un conteneur (détails complets)
docker inspect <id-conteneur>

# Voir les statistiques en temps réel
docker stats
```

## 🎯 Objectif

Mettre en place un serveur DNS et DHCP sur Windows Server pour permettre la résolution de noms et l'attribution automatique d'adresses IP sur un réseau local.

---

## 📋 Prérequis

|Élément|Valeur exemple|Pourquoi ?|
|---|---|---|
|**IP fixe du serveur**|172.16.10.5|Un serveur DNS/DHCP doit avoir une IP fixe|
|**Masque de sous-réseau**|255.255.255.0|Définit la taille du réseau (ici 254 machines)|
|**Nom de domaine**|wilder.lan|Zone DNS sur laquelle le serveur fait autorité|
|**Plage DHCP**|172.16.10.50 - 172.16.10.100|IPs distribuées automatiquement aux clients|
|**Passerelle**|172.16.10.1|Routeur du réseau|

---

## 🔧 PARTIE 1 : Configuration Réseau de Base

### Commandes PowerShell - Configuration IP

|Commande|Utilité|Quand l'utiliser|
|---|---|---|
|`Get-NetAdapter`|Lister les cartes réseau disponibles|Pour connaître le nom exact de ta carte (ex: Ethernet, Ethernet 2)|
|`Get-NetIPAddress`|Voir la configuration IP actuelle|Pour vérifier l'IP avant modification|
|`New-NetIPAddress -InterfaceAlias "Ethernet 2" -IPAddress 172.16.10.5 -PrefixLength 24 -DefaultGateway 172.16.10.1`|Configurer une IP fixe|Au tout début, pour donner une IP fixe au serveur|
|`Set-DnsClientServerAddress -InterfaceAlias "Ethernet 2" -ServerAddresses "127.0.0.1"`|Configurer le serveur DNS à utiliser|Le serveur doit pointer vers lui-même (127.0.0.1)|
|`Remove-NetIPAddress -InterfaceAlias "Ethernet 2" -IPAddress 172.16.10.2 -Confirm:$false`|Supprimer une ancienne IP|Si tu as configuré une mauvaise IP et veux la changer|
|`ipconfig /all`|Voir toute la configuration réseau|Pour vérifier que tout est bien configuré|
|`Test-NetConnection -ComputerName 172.16.10.5`|Tester la connectivité réseau|Pour vérifier que le serveur est joignable|

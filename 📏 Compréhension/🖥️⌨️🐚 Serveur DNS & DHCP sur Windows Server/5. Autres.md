## 🔄 Ordre d'exécution logique complet

### Phase 1 : Préparation du serveur

```
1. Get-NetAdapter                                    → Identifier la carte réseau
2. New-NetIPAddress -InterfaceAlias "Ethernet 2" -IPAddress 172.16.10.5 -PrefixLength 24 -DefaultGateway 172.16.10.1
3. Set-DnsClientServerAddress -InterfaceAlias "Ethernet 2" -ServerAddresses "127.0.0.1"
4. ipconfig /all                                     → Vérifier la configuration
```

### Phase 2 : Installation DNS

```
5. Install-WindowsFeature -Name DNS -IncludeManagementTools
6. Start-Service DNS
7. Get-Service DNS                                   → Vérifier que DNS est actif
```

### Phase 3 : Configuration DNS

```
8. Add-DnsServerPrimaryZone -Name "wilder.lan" -ZoneFile "wilder.lan.dns"
9. Add-DnsServerPrimaryZone -NetworkId "172.16.10.0/24" -ZoneFile "10.16.172.in-addr.arpa.dns"
10. Add-DnsServerResourceRecordA -Name "serveur" -ZoneName "wilder.lan" -IPv4Address "172.16.10.5" -CreatePtr
11. Add-DnsServerResourceRecordCName -Name "dns" -HostNameAlias "serveur.wilder.lan" -ZoneName "wilder.lan"
12. Get-DnsServerResourceRecord -ZoneName "wilder.lan"   → Vérifier les enregistrements
```

### Phase 4 : Tests DNS (sur le serveur)

```
13. nslookup serveur.wilder.lan
14. nslookup dns.wilder.lan
15. nslookup 172.16.10.5
16. ping serveur.wilder.lan
```

### Phase 5 : Installation DHCP

```
17. Install-WindowsFeature -Name DHCP -IncludeManagementTools
18. Restart-Service DHCPServer
19. Get-Service DHCPServer                           → Vérifier que DHCP est actif
```

### Phase 6 : Configuration DHCP

```
20. Add-DhcpServerv4Scope -Name "Atelier_LAN" -StartRange 172.16.10.50 -EndRange 172.16.10.100 -SubnetMask 255.255.255.0 -State Active
21. Set-DhcpServerv4OptionValue -ScopeId 172.16.10.0 -Router 172.16.10.1
22. Set-DhcpServerv4OptionValue -ScopeId 172.16.10.0 -DnsServer 172.16.10.5
23. Set-DhcpServerv4OptionValue -ScopeId 172.16.10.0 -DnsDomain "wilder.lan"
24. Get-DhcpServerv4OptionValue -ScopeId 172.16.10.0  → Vérifier les options
```

### Phase 7 : Tests depuis le CLIENT

```
25. ipconfig /release                                 → Libérer l'ancienne IP
26. ipconfig /renew                                   → Obtenir une IP du DHCP
27. ipconfig /all                                     → Vérifier : IP, DNS, Gateway
28. ipconfig /flushdns                                → Vider le cache DNS
29. ping 172.16.10.5                                  → Tester connectivité serveur
30. nslookup serveur.wilder.lan                       → Tester résolution DNS
31. ping serveur.wilder.lan                           → Tester résolution + connectivité
32. ping dns.wilder.lan                               → Tester l'alias CNAME
33. nslookup 172.16.10.5                              → Tester résolution inverse
```

### Phase 8 : Vérifications finales (sur le serveur)

```
34. Get-DhcpServerv4Lease -ScopeId 172.16.10.0       → Voir les clients connectés
35. Get-DnsServerResourceRecord -ZoneName "wilder.lan" → Voir tous les enregistrements DNS
```

## 🚨 Dépannage : Commandes de diagnostic

|Problème|Commande de diagnostic|Solution|
|---|---|---|
|DNS ne démarre pas|`Get-Service DNS` puis `Get-EventLog -LogName System -Source "DNS" -Newest 20`|Vérifier les logs d'erreur|
|DHCP ne distribue pas d'IP|`Get-DhcpServerv4Scope` puis vérifier State = Active|Activer l'étendue|
|Client ne résout pas les noms|`ipconfig /all` (vérifier DNS Servers)|Renouveler le bail DHCP ou configurer DNS manuellement|
|Pare-feu bloque|`Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False`|Désactiver temporairement pour tester|
|Enregistrement DNS incorrect|`Remove-DnsServerResourceRecord` puis recréer|Supprimer et recréer l'enregistrement|
|Cache DNS obsolète|`Clear-DnsClientCache` (client) ou `Clear-DnsServerCache` (serveur)|Vider les caches|

## ✅ Critères de validation

- [ ]  Le serveur a une IP fixe (172.16.10.5)
- [ ]  Le service DNS est actif et fonctionne
- [ ]  La zone DNS directe (wilder.lan) existe avec au moins 1 enregistrement A
- [ ]  La zone DNS inverse existe avec au moins 1 enregistrement PTR
- [ ]  Au moins 1 enregistrement CNAME est configuré
- [ ]  Le service DHCP est actif et distribue des IP
- [ ]  Les options DHCP (Gateway, DNS) sont configurées
- [ ]  Le client obtient automatiquement une IP du DHCP
- [ ]  Le client peut résoudre `serveur.wilder.lan` → 172.16.10.5
- [ ]  Le client peut résoudre `dns.wilder.lan` → 172.16.10.5 (via CNAME)
- [ ]  Le client peut faire une résolution inverse de 172.16.10.5 → serveur.wilder.lan
- [ ]  Le ping fonctionne entre client et serveur avec les noms DNS

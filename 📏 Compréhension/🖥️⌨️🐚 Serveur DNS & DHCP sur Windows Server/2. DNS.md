## 🌐 PARTIE 2 : Installation et Configuration DNS

### Étape 1 : Installation du rôle DNS

|Commande|Utilité|
|---|---|
|`Install-WindowsFeature -Name DNS -IncludeManagementTools`|Installer le rôle DNS avec les outils d'administration|
|`Get-WindowsFeature DNS`|Vérifier que DNS est installé|
|`Get-Service DNS`|Vérifier l'état du service DNS|
|`Start-Service DNS`|Démarrer le service DNS|
|`Restart-Service DNS`|Redémarrer le service DNS après modification|

### Étape 2 : Création de la zone DNS directe (nom → IP)

**Via l'interface graphique :**

1. Ouvrir **DNS Manager** (Tools → DNS)
2. Clic droit sur **Forward Lookup Zones** → New Zone
3. Primary zone → Next
4. Zone name : `wilder.lan` → Next → Finish

**Via PowerShell :**

|Commande|Utilité|
|---|---|
|`Add-DnsServerPrimaryZone -Name "wilder.lan" -ZoneFile "wilder.lan.dns"`|Créer une zone DNS directe|
|`Get-DnsServerZone`|Lister toutes les zones DNS configurées|

### Étape 3 : Création de la zone DNS inverse (IP → nom)

**Via l'interface graphique :**

1. Clic droit sur **Reverse Lookup Zones** → New Zone
2. Primary zone → Next
3. IPv4 → Next
4. Network ID : `172.16.10` → Next → Finish

**Via PowerShell :**

|Commande|Utilité|
|---|---|
|`Add-DnsServerPrimaryZone -NetworkId "172.16.10.0/24" -ZoneFile "10.16.172.in-addr.arpa.dns"`|Créer une zone DNS inverse|

### Étape 4 : Création des enregistrements DNS

|Commande|Utilité|Type d'enregistrement|
|---|---|---|
|`Add-DnsServerResourceRecordA -Name "serveur" -ZoneName "wilder.lan" -IPv4Address "172.16.10.5" -CreatePtr`|Créer un enregistrement A (nom → IP) avec PTR automatique|**A** (Address)|
|`Add-DnsServerResourceRecordCName -Name "dns" -HostNameAlias "serveur.wilder.lan" -ZoneName "wilder.lan"`|Créer un alias (CNAME)|**CNAME** (Canonical Name)|
|`Add-DnsServerResourceRecordPtr -Name "5" -ZoneName "10.16.172.in-addr.arpa" -PtrDomainName "serveur.wilder.lan"`|Créer manuellement un enregistrement PTR (IP → nom)|**PTR** (Pointer)|
|`Get-DnsServerResourceRecord -ZoneName "wilder.lan"`|Lister tous les enregistrements d'une zone|Vérification|
|`Remove-DnsServerResourceRecord -ZoneName "wilder.lan" -Name "serveur" -RRType A`|Supprimer un enregistrement|Correction d'erreur|

### Étape 5 : Tests DNS

|Commande|Utilité|Résultat attendu|
|---|---|---|
|`nslookup serveur.wilder.lan`|Tester la résolution directe (nom → IP)|Address: 172.16.10.5|
|`nslookup dns.wilder.lan`|Tester l'alias CNAME|Address: 172.16.10.5 (via serveur.wilder.lan)|
|`nslookup 172.16.10.5`|Tester la résolution inverse (IP → nom)|Name: serveur.wilder.lan|
|`Resolve-DnsName serveur.wilder.lan`|Alternative PowerShell à nslookup|IP4Address: 172.16.10.5|
|`Test-DnsServer -IPAddress 172.16.10.5 -ZoneName "wilder.lan"`|Tester le serveur DNS complet|RR Query: Success|
|`ping serveur.wilder.lan`|Tester la résolution + connectivité|Reply from 172.16.10.5|
|`Clear-DnsClientCache`|Vider le cache DNS du client|Après modification d'enregistrements|

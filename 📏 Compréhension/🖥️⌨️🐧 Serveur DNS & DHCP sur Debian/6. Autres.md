## 🔄 Ordre d'exécution logique complet (Debian)

### Phase 1 : Préparation du serveur

```bash
1. ip addr show                                           # Identifier les interfaces
2. sudo nano /etc/network/interfaces                      # Configurer IP fixe
   # Ajouter la config pour enp0s8 avec IP 192.168.10.5
3. sudo systemctl restart networking                      # Appliquer la config
   # OU: sudo reboot
4. ip addr show                                           # Vérifier la config
5. sudo hostnamectl set-hostname serveur-debian           # Nommer le serveur
```

### Phase 2 : Installation des services

```bash
6. sudo apt update                                        # Mettre à jour les paquets
7. sudo apt install bind9 bind9-utils bind9-doc -y       # Installer Bind9
8. sudo apt install isc-dhcp-server -y                    # Installer DHCP
9. dpkg -l | grep -E 'bind9|isc-dhcp-server'             # Vérifier installations
```

### Phase 3 : Configuration DNS (Bind9)

```bash
10. sudo nano /etc/bind/named.conf.options                # Config options globales
    # Ajouter listen-on, allow-query, recursion, forwarders
11. sudo nano /etc/bind/named.conf.local                  # Déclarer les zones
    # Ajouter zone directe et inverse
12. sudo nano /etc/bind/db.wilders.debian.lan             # Créer zone directe
    # Ajouter SOA, NS, A, CNAME
13. sudo nano /etc/bind/db.192.168.10                     # Créer zone inverse
    # Ajouter SOA, NS, PTR
14. sudo named-checkconf                                  # Vérifier syntaxe générale
15. sudo named-checkzone wilders.debian.lan /etc/bind/db.wilders.debian.lan
16. sudo named-checkzone 10.168.192.in-addr.arpa /etc/bind/db.192.168.10
17. sudo systemctl restart bind9                          # Redémarrer DNS
18. sudo systemctl enable bind9                           # Activer au démarrage
19. sudo systemctl status bind9                           # Vérifier le statut
```

### Phase 4 : Tests DNS (sur le serveur)

```bash
20. dig @127.0.0.1 serveur.wilders.debian.lan             # Test résolution A
21. dig @127.0.0.1 dns.wilders.debian.lan                 # Test CNAME
22. dig @127.0.0.1 -x 192.168.10.5                        # Test résolution inverse
23. nslookup serveur.wilders.debian.lan 127.0.0.1        # Alternative nslookup
24. ping -c 4 serveur.wilders.debian.lan                  # Test connectivité
```

### Phase 5 : Configuration DHCP

```bash
25. sudo nano /etc/default/isc-dhcp-server                # Spécifier interface
    # INTERFACESv4="enp0s8"
26. sudo nano /etc/dhcp/dhcpd.conf                        # Configurer DHCP
    # Ajouter subnet, range, options
27. sudo dhcpd -t -cf /etc/dhcp/dhcpd.conf               # Vérifier syntaxe
28. sudo systemctl restart isc-dhcp-server                # Redémarrer DHCP
29. sudo systemctl enable isc-dhcp-server                 # Activer au démarrage
30. sudo systemctl status isc-dhcp-server                 # Vérifier le statut
```

### Phase 6 : Tests depuis le CLIENT

```bash
31. sudo dhclient -r enp0s8                               # Libérer IP (Linux)
    # OU: ipconfig /release (Windows)
32. sudo dhclient enp0s8                                  # Obtenir nouvelle IP
    # OU: ipconfig /renew (Windows)
33. ip addr show                                          # Vérifier IP reçue
    # OU: ipconfig /all (Windows)
34. cat /etc/resolv.conf                                  # Vérifier DNS configuré
35. ping -c 4 192.168.10.5                                # Test connectivité serveur
36. ping -c 4 serveur.wilders.debian.lan                  # Test résolution + ping
37. nslookup serveur.wilders.debian.lan                   # Test résolution DNS
38. nslookup dns.wilders.debian.lan                       # Test alias CNAME
39. nslookup 192.168.10.5                                 # Test résolution inverse
```

### Phase 7 : Vérifications finales (sur le serveur)

```bash
40. sudo cat /var/lib/dhcp/dhcpd.leases                   # Voir baux DHCP actifs
41. sudo journalctl -xeu bind9                            # Vérifier logs DNS
42. sudo journalctl -xeu isc-dhcp-server                  # Vérifier logs DHCP
```

## 🚨 Dépannage : Commandes de diagnostic

### Diagnostic DNS

|Problème|Commande de diagnostic|Solution|
|---|---|---|
|Bind9 ne démarre pas|`sudo systemctl status bind9`<br>`sudo journalctl -xeu bind9`|Vérifier syntaxe avec `named-checkconf`|
|Zone DNS mal configurée|`sudo named-checkzone wilders.debian.lan /etc/bind/db.wilders.debian.lan`|Corriger les erreurs de syntaxe|
|Port 53 occupé|`sudo netstat -tulpn \| grep :53`<br>`sudo ss -tulpn \| grep :53`|Identifier le processus et l'arrêter|
|Permissions fichiers|`ls -l /etc/bind/db.*`|`sudo chown bind:bind /etc/bind/db.*`|
|Pare-feu bloque|`sudo ufw status`|`sudo ufw allow 53/tcp`<br>`sudo ufw allow 53/udp`|

### Diagnostic DHCP

|Problème|Commande de diagnostic|Solution|
|---|---|---|
|DHCP ne démarre pas|`sudo systemctl status isc-dhcp-server`<br>`sudo journalctl -xeu isc-dhcp-server`|Vérifier syntaxe avec `dhcpd -t`|
|Interface mal configurée|`sudo cat /etc/default/isc-dhcp-server`|Vérifier `INTERFACESv4="enp0s8"`|
|Port 67 occupé|`sudo netstat -ulpn \| grep :67`<br>`sudo ss -ulpn \| grep :67`|Arrêter autre serveur DHCP|
|Pas de baux distribués|`sudo tail -f /var/log/syslog \| grep dhcpd`|Vérifier que l'étendue est active|
|Pare-feu bloque|`sudo ufw status`|`sudo ufw allow 67/udp`<br>`sudo ufw allow 68/udp`|

### Diagnostic Réseau

|Problème|Commande|Solution|
|---|---|---|
|Interface DOWN|`ip link show enp0s8`|`sudo ip link set enp0s8 up`|
|Pas de route par défaut|`ip route show`|Ajouter gateway dans `/etc/network/interfaces`|
|DNS non résolu|`cat /etc/resolv.conf`|Vérifier `nameserver 192.168.10.5`|
|Cache DNS obsolète|`sudo systemd-resolve --flush-caches`|Vider cache système|

## 📊 Récapitulatif des ports et protocoles

|Service|Port|Protocole|Utilité|
|---|---|---|---|
|DNS|53|TCP/UDP|Résolution de noms de domaine|
|DHCP Server|67|UDP|Serveur écoute les demandes DHCP|
|DHCP Client|68|UDP|Client envoie les demandes DHCP|

**Commandes pour ouvrir les ports (si pare-feu actif) :**

```bash
# UFW (Uncomplicated Firewall)
sudo ufw allow 53/tcp
sudo ufw allow 53/udp
sudo ufw allow 67/udp
sudo ufw allow 68/udp

# iptables (alternative)
sudo iptables -A INPUT -p tcp --dport 53 -j ACCEPT
sudo iptables -A INPUT -p udp --dport 53 -j ACCEPT
sudo iptables -A INPUT -p udp --dport 67 -j ACCEPT
```

```
├── cache/bind/                 # Cache DNS
├── lib/dhcp/

└── log/
    └── syslog                  # Logs système (DNS + DHCP)
```

## 🌐 PARTIE 3 : Configuration DNS (Bind9)

### Gestion du service Bind9

|Commande|Utilité|
|---|---|
|`sudo systemctl status bind9`|Voir l'état du service DNS|
|`sudo systemctl start bind9`|Démarrer le service DNS|
|`sudo systemctl stop bind9`|Arrêter le service DNS|
|`sudo systemctl restart bind9`|Redémarrer le service DNS|
|`sudo systemctl enable bind9`|Activer le démarrage automatique au boot|
|`sudo systemctl disable bind9`|Désactiver le démarrage automatique|
|`sudo journalctl -xeu bind9`|Voir les logs d'erreur du service|
|`sudo tail -f /var/log/syslog \| grep named`|Suivre les logs DNS en temps réel|

### Fichiers de configuration Bind9

|Fichier|Rôle|
|---|---|
|`/etc/bind/named.conf`|Fichier principal (inclut les autres)|
|`/etc/bind/named.conf.options`|Options globales du serveur DNS|
|`/etc/bind/named.conf.local`|Déclaration des zones DNS locales|
|`/etc/bind/db.wilders.debian.lan`|Fichier de zone directe (nom → IP)|
|`/etc/bind/db.192.168.10`|Fichier de zone inverse (IP → nom)|

### Étape 1 : Configuration des options Bind9

**Fichier :** `sudo nano /etc/bind/named.conf.options`

```bash
options {
    directory "/var/cache/bind";
    
    # Écoute sur localhost et sur l'IP du serveur
    listen-on { 127.0.0.1; 192.168.10.5; };
    
    # Autoriser les requêtes du réseau local
    allow-query { localhost; 192.168.10.0/24; };
    
    # Activer la récursion pour le réseau local
    recursion yes;
    allow-recursion { localhost; 192.168.10.0/24; };
    
    # Forwarders (serveurs DNS externes pour Internet)
    forwarders {
        8.8.8.8;
        8.8.4.4;
    };
    
    dnssec-validation auto;
    listen-on-v6 { any; };
};
```

### Étape 2 : Déclaration des zones DNS

**Fichier :** `sudo nano /etc/bind/named.conf.local`

```bash
# Déclaration de la zone DNS

# Zone de recherche directe (nom -> IP)
zone "wilders.debian.lan" {
    type master;
    file "/etc/bind/db.wilders.debian.lan";
};

# Zone de recherche inverse (IP -> nom)
zone "10.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/db.192.168.10";
};
```

### Étape 3 : Création du fichier de zone directe

**Fichier :** `sudo nano /etc/bind/db.wilders.debian.lan`

```bash
;
; Fichier de zone pour wilders.debian.lan
;
$TTL    604800
@       IN      SOA     ns.wilders.debian.lan. admin.wilders.debian.lan. (
                              2         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
;
; Serveurs de noms
@       IN      NS      ns.wilders.debian.lan.

; Enregistrements A (nom -> IP)
ns              IN      A       192.168.10.5
serveur         IN      A       192.168.10.5

; Enregistrements CNAME (alias)
dns             IN      CNAME   serveur.wilders.debian.lan.
www             IN      CNAME   serveur.wilders.debian.lan.
```

**Explication des champs :**

- `$TTL` : Durée de vie des enregistrements en cache (604800 = 7 jours)
- `SOA` : Start of Authority (infos sur la zone)
- `NS` : Name Server (serveur DNS faisant autorité)
- `A` : Enregistrement principal (nom → IP)
- `CNAME` : Alias (un nom pointe vers un autre nom)
- **`;`** : Commentaire (obligatoire dans les fichiers de zone, pas `#`)

### Étape 4 : Création du fichier de zone inverse

**Fichier :** `sudo nano /etc/bind/db.192.168.10`

```bash
;
; Fichier de zone inverse pour 192.168.10.0/24
;
$TTL    604800
@       IN      SOA     ns.wilders.debian.lan. admin.wilders.debian.lan. (
                              2         ; Serial
                         604800         ; Refresh
                          86400         ; Retry
                        2419200         ; Expire
                         604800 )       ; Negative Cache TTL
;
@       IN      NS      ns.wilders.debian.lan.

; Enregistrements PTR (IP -> nom)
5       IN      PTR     serveur.wilders.debian.lan.
```

**Explication :**

- `5 IN PTR` : L'IP 192.168.10.**5** pointe vers serveur.wilders.debian.lan

### Étape 5 : Vérification de la syntaxe

|Commande|Utilité|
|---|---|
|`sudo named-checkconf`|Vérifier la syntaxe de `named.conf.*`|
|`sudo named-checkzone wilders.debian.lan /etc/bind/db.wilders.debian.lan`|Vérifier la zone directe|
|`sudo named-checkzone 10.168.192.in-addr.arpa /etc/bind/db.192.168.10`|Vérifier la zone inverse|

**Résultat attendu :** `OK` ou `zone ... loaded serial ...`

### Étape 6 : Redémarrer Bind9

```bash
sudo systemctl restart bind9
sudo systemctl status bind9
```

### Tests DNS depuis le serveur

|Commande|Utilité|Résultat attendu|
|---|---|---|
|`dig @127.0.0.1 serveur.wilders.debian.lan`|Test résolution directe avec dig|ANSWER: serveur.wilders.debian.lan. IN A 192.168.10.5|
|`dig @127.0.0.1 dns.wilders.debian.lan`|Test alias CNAME avec dig|ANSWER: dns → serveur → 192.168.10.5|
|`dig @127.0.0.1 -x 192.168.10.5`|Test résolution inverse avec dig|ANSWER: 5.10.168.192.in-addr.arpa. IN PTR serveur...|
|`nslookup serveur.wilders.debian.lan 127.0.0.1`|Test résolution avec nslookup|Address: 192.168.10.5|
|`nslookup dns.wilders.debian.lan 127.0.0.1`|Test alias avec nslookup|canonical name = serveur...|
|`nslookup 192.168.10.5 127.0.0.1`|Test résolution inverse|name = serveur.wilders.debian.lan|
|`host serveur.wilders.debian.lan 127.0.0.1`|Test avec commande host|has address 192.168.10.5|
|`ping -c 4 serveur.wilders.debian.lan`|Test résolution + connectivité|64 bytes from 192.168.10.5|

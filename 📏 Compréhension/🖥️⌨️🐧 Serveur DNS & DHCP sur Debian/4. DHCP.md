## 📡 PARTIE 4 : Configuration DHCP (ISC-DHCP-Server)

### Gestion du service DHCP

|Commande|Utilité|
|---|---|
|`sudo systemctl status isc-dhcp-server`|Voir l'état du service DHCP|
|`sudo systemctl start isc-dhcp-server`|Démarrer le service DHCP|
|`sudo systemctl stop isc-dhcp-server`|Arrêter le service DHCP|
|`sudo systemctl restart isc-dhcp-server`|Redémarrer le service DHCP|
|`sudo systemctl enable isc-dhcp-server`|Activer le démarrage automatique|
|`sudo journalctl -xeu isc-dhcp-server`|Voir les logs d'erreur du service|
|`sudo tail -f /var/log/syslog \| grep dhcpd`|Suivre les logs DHCP en temps réel|

### Fichiers de configuration DHCP

|Fichier|Rôle|
|---|---|
|`/etc/dhcp/dhcpd.conf`|Configuration principale du serveur DHCP|
|`/etc/default/isc-dhcp-server`|Spécifier sur quelle interface écouter|
|`/var/lib/dhcp/dhcpd.leases`|Fichier des baux DHCP actifs (généré automatiquement)|

### Étape 1 : Configuration de l'interface d'écoute

**Fichier :** `sudo nano /etc/default/isc-dhcp-server`

```bash
# Defaults for isc-dhcp-server (sourced by /etc/init.d/isc-dhcp-server)

# Path to dhcpd's config file (default: /etc/dhcp/dhcpd.conf).
#DHCPDv4_CONF=/etc/dhcp/dhcpd.conf
#DHCPDv6_CONF=/etc/dhcp/dhcpd6.conf

# Path to dhcpd's PID file (default: /var/run/dhcpd.pid).
#DHCPDv4_PID=/var/run/dhcpd.pid
#DHCPDv6_PID=/var/run/dhcpd6.pid

# Additional options to start dhcpd with.
#   Don't use options -cf or -pf here; use DHCPD_CONF/ DHCPD_PID instead
#OPTIONS=""

# On what interfaces should the DHCP server (dhcpd) serve DHCP requests?
#   Separate multiple interfaces with spaces, e.g. "eth0 eth1".
INTERFACESv4="enp0s8"
INTERFACESv6=""
```

**Important :** Remplace `enp0s8` par le nom de ton interface réseau interne.

### Étape 2 : Configuration du serveur DHCP

**Fichier :** `sudo nano /etc/dhcp/dhcpd.conf`

**Configuration complète :**

```bash
# Configuration globale
option domain-name "wilders.debian.lan";
option domain-name-servers 192.168.10.5;
default-lease-time 600;
max-lease-time 7200;
authoritative;

# Sous-réseau 192.168.10.0/24
subnet 192.168.10.0 netmask 255.255.255.0 {
    range 192.168.10.50 192.168.10.100;
    option routers 192.168.10.1;
    option subnet-mask 255.255.255.0;
    option domain-name-servers 192.168.10.5;
    option domain-name "wilders.debian.lan";
}
```

**Explication ligne par ligne :**

- `option domain-name` : Nom de domaine par défaut
- `option domain-name-servers` : Serveur DNS à utiliser (TON serveur)
- `default-lease-time` : Durée du bail DHCP (600s = 10 min)
- `max-lease-time` : Durée maximale du bail (7200s = 2h)
- `authoritative` : Ce serveur fait autorité sur ce réseau
- `subnet ... netmask` : Définition du sous-réseau
- `range` : Plage d'IP à distribuer (50 à 100 = 50 machines)
- `option routers` : Passerelle par défaut
- `option subnet-mask` : Masque de sous-réseau

### Étape 3 : Vérification de la syntaxe

```bash
sudo dhcpd -t -cf /etc/dhcp/dhcpd.conf
```

**Résultat attendu :** Aucune erreur ou juste des warnings sans importance

### Étape 4 : Redémarrer le service DHCP

```bash
sudo systemctl restart isc-dhcp-server
sudo systemctl status isc-dhcp-server
```

**Résultat attendu :** `active (running)` en vert

### Gestion des baux DHCP

|Commande|Utilité|
|---|---|
|`sudo cat /var/lib/dhcp/dhcpd.leases`|Voir tous les baux DHCP|
|`sudo grep "lease" /var/lib/dhcp/dhcpd.leases`|Voir uniquement les IP louées|
|`sudo dhcp-lease-list`|Lister les baux actifs (si installé)|
|`sudo systemctl restart isc-dhcp-server`|Réinitialiser les baux|

### Réservation DHCP (IP fixe pour un client)

**Ajouter dans `/etc/dhcp/dhcpd.conf` :**

```bash
# Réservation pour un client spécifique
host windows10 {
    hardware ethernet 08:00:27:0e:78:87;
    fixed-address 192.168.10.10;
}
```

**Puis redémarrer :**

```bash
sudo systemctl restart isc-dhcp-server
```

# 📘 Fiche de Révision : Serveur DNS & DHCP sur Debian/Linux

## 🎯 Objectif

Mettre en place un serveur DNS (Bind9) et DHCP (ISC-DHCP-Server) sur Debian pour permettre la résolution de noms et l'attribution automatique d'adresses IP sur un réseau local.

---

## 📋 Prérequis

|Élément|Valeur exemple|Pourquoi ?|
|---|---|---|
|**IP fixe du serveur**|192.168.10.5|Un serveur DNS/DHCP doit avoir une IP fixe|
|**Masque de sous-réseau**|255.255.255.0 (/24)|Définit la taille du réseau (254 machines)|
|**Nom de domaine**|wilders.debian.lan|Zone DNS sur laquelle le serveur fait autorité|
|**Plage DHCP**|192.168.10.50 - 192.168.10.100|IPs distribuées automatiquement aux clients|
|**Passerelle**|192.168.10.1|Routeur du réseau|
|**Interface réseau**|enp0s8|Interface réseau du réseau interne|

---

## 🔧 PARTIE 1 : Configuration Réseau de Base

### Commandes Bash - Configuration IP

|Commande|Utilité|Quand l'utiliser|
|---|---|---|
|`ip addr show` ou `ip a`|Lister toutes les interfaces réseau et leurs IP|Pour voir la configuration actuelle|
|`ip link show`|Voir l'état des interfaces (UP/DOWN)|Pour vérifier si une interface est active|
|`ip route show`|Afficher la table de routage|Pour voir quelle est la route par défaut|
|`sudo nano /etc/network/interfaces`|Éditer le fichier de configuration réseau|Pour configurer une IP fixe|
|`sudo systemctl restart networking`|Redémarrer le service réseau|Après modification du fichier interfaces|
|`sudo reboot`|Redémarrer la machine|Alternative au restart networking|
|`ping -c 4 192.168.10.5`|Tester la connectivité (4 paquets)|Pour vérifier que le serveur répond|
|`hostname`|Voir le nom d'hôte actuel|Pour vérifier l'identité de la machine|
|`sudo hostnamectl set-hostname serveur-debian`|Changer le nom d'hôte|Pour donner un nom significatif au serveur|

### Configuration IP fixe dans `/etc/network/interfaces`

**Fichier à éditer :** `sudo nano /etc/network/interfaces`

```bash
# Interface loopback
auto lo
iface lo inet loopback

# Interface réseau interne (IP fixe)
auto enp0s8
iface enp0s8 inet static
    address 192.168.10.5
    netmask 255.255.255.0
    # gateway 192.168.10.1    # Pas nécessaire si NAT sur autre interface
    dns-nameservers 127.0.0.1

# Interface NAT (optionnelle, pour accès Internet)
allow-hotplug enp0s3
iface enp0s3 inet dhcp
```

**Appliquer les changements :**

```bash
sudo systemctl restart networking
# OU
sudo reboot
```

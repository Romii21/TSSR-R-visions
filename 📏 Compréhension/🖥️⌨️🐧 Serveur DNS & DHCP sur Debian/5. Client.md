## 💻 PARTIE 5 : Commandes Côté CLIENT

### Configuration réseau du client

**Sur un client Linux :**

|Commande|Utilité|
|---|---|
|`ip addr show`|Voir la configuration IP actuelle|
|`sudo dhclient -r enp0s8`|Libérer le bail DHCP (release)|
|`sudo dhclient enp0s8`|Demander un nouveau bail DHCP (renew)|
|`sudo systemctl restart NetworkManager`|Redémarrer le gestionnaire réseau|
|`cat /etc/resolv.conf`|Voir le serveur DNS configuré|
|`resolvectl status`|Voir la configuration DNS détaillée|

**Sur un client Windows :**

|Commande|Utilité|
|---|---|
|`ipconfig /all`|Voir toute la configuration réseau|
|`ipconfig /release`|Libérer l'adresse IP|
|`ipconfig /renew`|Demander une nouvelle IP|
|`ipconfig /flushdns`|Vider le cache DNS|

### Tests depuis le client

|Commande|Utilité|Résultat attendu|
|---|---|---|
|`ip addr show`|Vérifier l'IP obtenue|inet 192.168.10.XX/24|
|`cat /etc/resolv.conf`|Vérifier le serveur DNS|nameserver 192.168.10.5|
|`ping -c 4 192.168.10.5`|Tester connectivité serveur|64 bytes from 192.168.10.5|
|`ping -c 4 serveur.wilders.debian.lan`|Tester résolution + connectivité|64 bytes from 192.168.10.5|
|`dig serveur.wilders.debian.lan`|Tester résolution DNS détaillée|ANSWER: 192.168.10.5|
|`nslookup serveur.wilders.debian.lan`|Tester résolution DNS simple|Address: 192.168.10.5|
|`nslookup dns.wilders.debian.lan`|Tester alias CNAME|canonical name = serveur...|
|`nslookup 192.168.10.5`|Tester résolution inverse|name = serveur.wilders.debian.lan|

## IPv6 : Conversions

### Structure d'une adresse IPv6

Une adresse IPv6 est composée de **8 groupes de 16 bits** (128 bits au total), écrits en hexadécimal :

```
2001:0db8:85a3:0000:0000:8a2e:0370:7334
```

### Règles de compression IPv6

1. **Suppression des zéros de tête** dans chaque groupe
2. **Remplacement d'un ou plusieurs groupes de zéros consécutifs** par `::` (une seule fois)

**Exemple de compression :**

```
Adresse complète : 2001:0db8:0000:0000:0000:0000:0000:0001
Compression étape 1 : 2001:db8:0:0:0:0:0:1
Compression finale : 2001:db8::1
```

**Autre exemple :**

```
Adresse complète : fe80:0000:0000:0000:0204:61ff:fe9d:f156
Compressée : fe80::204:61ff:fe9d:f156
```

### Conversion hexadécimal ↔ binaire (IPv6)

**Exemple : 2001:db8::1**

**Décompression complète :**

```
2001:0db8:0000:0000:0000:0000:0000:0001
```

**Conversion en binaire :**

- `2001` → `0010 0000 0000 0001`
- `0db8` → `0000 1101 1011 1000`
- `0000` → `0000 0000 0000 0000`
- `0001` → `0000 0000 0000 0001`

**Résultat complet (128 bits) :**

```
0010000000000001 0000110110111000 0000000000000000 0000000000000000
0000000000000000 0000000000000000 0000000000000000 0000000000000001
```

### Types d'adresses IPv6 courantes

|Type|Préfixe|Exemple|Usage|
|---|---|---|---|
|Unicast globale|2000::/3|2001:db8::1|Internet|
|Link-local|fe80::/10|fe80::1|Réseau local uniquement|
|Loopback|::1/128|::1|Localhost (équivalent 127.0.0.1)|
|Multicast|ff00::/8|ff02::1|Communication de groupe|
|Documentation|2001:db8::/32|2001:db8::1|Exemples/documentation|

### Notation CIDR IPv6

Les masques IPv6 utilisent aussi la notation CIDR :

- `/64` : taille standard pour un sous-réseau (préfixe de 64 bits)
- `/128` : adresse d'hôte unique
- `/48` : taille d'allocation typique pour une organisation

**Exemple : 2001:db8:abcd::/48**

- Préfixe réseau : 2001:db8:abcd
- Peut être subdivisé en 65 536 sous-réseaux /64

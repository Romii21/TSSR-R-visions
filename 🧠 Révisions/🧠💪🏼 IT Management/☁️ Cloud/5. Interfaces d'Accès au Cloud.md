### 5.1 SSH (Secure Shell)

**Description** : Accès sécurisé en ligne de commande pour la gestion des serveurs

**Commande de connexion :**

```bash
ssh utilisateur@adresse-ip-serveur
# Exemple
ssh admin@192.168.1.100

# Avec clé privée
ssh -i ~/.ssh/cle_privee.pem ubuntu@ec2-instance.amazonaws.com
```

**Avantages :**

- Contrôle complet du serveur
- Automatisation possible via scripts

**Inconvénients :**

- Nécessite des compétences techniques
- Moins intuitif pour les débutants

### 5.2 Interface Web (Console)

**Description** : Accès via navigateur web, interface graphique

**Exemples :**

- AWS Console : https://console.aws.amazon.com
- Azure Portal : https://portal.azure.com
- Google Cloud Console : https://console.cloud.google.com

**Avantages :**

- Facile et intuitif
- Pas de configuration locale nécessaire
- Idéal pour la gestion de base

**Inconvénients :**

- Moins adapté pour l'automatisation
- Peut être limité pour des opérations complexes

### 5.3 API (Application Programming Interface)

**Description** : Permet l'intégration et l'automatisation programmables

**Exemple avec AWS CLI :**

```bash
# Lister les instances EC2
aws ec2 describe-instances

# Créer un bucket S3
aws s3 mb s3://mon-bucket

# Copier un fichier vers S3
aws s3 cp fichier.txt s3://mon-bucket/
```

**Exemple avec Azure CLI :**

```bash
# Lister les VMs
az vm list --output table

# Créer un groupe de ressources
az group create --name MonGroupe --location francecentral

# Créer une VM
az vm create --resource-group MonGroupe --name MaVM --image UbuntuLTS
```

**Avantages :**

- Automatisation complète
- Intégration dans des workflows
- Scripts reproductibles

### 5.4 Diagnostic et troubleshooting

#### Problème : "Je ne peux pas me connecter à ma VM"

**Vérifications :**

```bash
# 1. Vérifier la connectivité réseau
ping adresse-ip-vm

# 2. Tester le port SSH
telnet adresse-ip-vm 22

# 3. Tester RDP pour Windows
telnet adresse-ip-vm 3389

# 4. Vérifier les règles de pare-feu (exemple Azure)
az network nsg rule list --resource-group MonGroupe --nsg-name MonNSG --output table
```

**Solutions :**

- Vérifier les groupes de sécurité (Security Groups)
- Vérifier l'assignation d'IP publique
- Vérifier les règles de pare-feu

#### Problème : "Mon application est lente"

**Diagnostic :**

- Consulter les métriques CPU/RAM de l'hyperviseur
- Vérifier la bande passante réseau (throttling ?)
- Analyser les logs applicatifs (Azure Monitor, CloudWatch)

#### Problème : "Les emails ne partent pas depuis M365"

**Vérifications :**

```bash
# Vérifier les enregistrements DNS
nslookup -type=MX domaine.fr

# Vérifier SPF
nslookup -type=TXT domaine.fr

# Vérifier DKIM
nslookup -type=TXT selector._domainkey.domaine.fr
```

**Solutions :**

- Vérifier les enregistrements DNS (SPF, DKIM, MX)
- Vérifier le Centre de messages M365 pour incidents
- Consulter les logs de transport Exchange

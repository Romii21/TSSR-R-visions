### 3.1 Niveaux de service Cloud

Le Cloud Computing se décline en plusieurs modèles de service selon le niveau de contrôle et de responsabilité.

#### Tableau comparatif des modèles de service

|Modèle|Acronyme|Description|Responsabilité fournisseur|Exemples|
|---|---|---|---|---|
|**On-Premises**|-|Infrastructure hébergée localement|Aucune|Serveurs internes de l'entreprise|
|**Infrastructure as a Service**|**IaaS**|Ressources informatiques virtualisées|Virtualisation, serveurs, stockage, réseau|AWS EC2, Azure VM, Google Compute Engine|
|**Platform as a Service**|**PaaS**|Environnement de développement et déploiement|IaaS + Runtime, middleware, OS|Azure App Service, Google App Engine, Heroku|
|**Software as a Service**|**SaaS**|Logiciels en ligne par abonnement|Tout (infrastructure + application)|Office 365, Salesforce, Gmail|

#### Explication des acronymes

- **IaaS** (Infrastructure as a Service) : Le fournisseur gère l'infrastructure physique (serveurs, stockage, réseau)
- **PaaS** (Platform as a Service) : Le fournisseur gère également l'environnement d'exécution (OS, middleware)
- **SaaS** (Software as a Service) : Le fournisseur gère tout, y compris l'application

### 3.2 Partage des responsabilités

![Schéma de responsabilités partagées]

- **IaaS** : Le fournisseur gère à partir de la virtualisation
- **PaaS** : Le fournisseur gère à partir du runtime/middleware
- **SaaS** : Le fournisseur gère tout (sauf les données clients)

⚠️ **Important** : Même dans le Cloud, les données nécessitent toujours des sauvegardes gérées par le client !

### 3.3 Fondation technique : La virtualisation

#### Hyperviseurs

Les hyperviseurs sont le socle de la virtualisation Cloud :

|Type|Description|Exemples|Usage|
|---|---|---|---|
|**Type 1 (Bare Metal)**|S'exécute directement sur le matériel physique|ESXi, Hyper-V, Proxmox, Xen, KVM|Datacenters cloud (AWS, Azure)|
|**Type 2 (Hosted)**|S'exécute sur un OS hôte|VirtualBox, VMware Workstation|Tests et développement|

**Équation du Cloud** :

```
Cloud = Virtualisation + Automatisation + Réseau
```

Les instances cloud (comme EC2 sur AWS) sont des machines virtuelles gérées automatiquement.

### 3.4 Modèles de déploiement

#### Cloud Public

**Caractéristiques :**

- Ressources partagées via Internet
- Multi-tenant (plusieurs clients sur la même infrastructure)
- Évolutif et économique

**Exemples :** AWS, Microsoft Azure, Google Cloud Platform

#### Cloud Privé

**Caractéristiques :**

- Ressources dédiées à une seule organisation
- Contrôle et sécurité renforcés
- Géré au sein de l'infrastructure de l'organisation

**Usage :** Grandes entreprises avec besoins de sécurité élevés

#### Cloud Hybride

**Caractéristiques :**

- Combinaison de cloud public et privé
- Flexibilité et optimisation des coûts
- Portabilité des données et applications

**Usage :** Applications critiques en privé, charges variables en public

### 3.5 Types d'hébergement

|Type|Description|Avantages|Limites|
|---|---|---|---|
|**Hébergement mutualisé**|Serveur partagé entre plusieurs clients|Coût réduit|Performances et personnalisation limitées|
|**VPS** (Virtual Private Server)|Serveur virtuel dédié par client|Plus de contrôle que le mutualisé|Limité par les ressources de l'hôte|
|**Serveur dédié** (Bare Metal)|Serveur physique complet à un client|Performances maximales|Coût élevé, moins flexible|

### 3.6 Types de stockage

#### Stockage Bloc (Block Storage)

- Disque virtuel attaché à une VM
- Usage : OS, bases de données, applications
- Performance élevée
- **Principe** : 1 disque = 1 VM

#### Stockage Objet (Object Storage)

- Stockage de fichiers (photos, vidéos, logs, backups)
- Pas de système de fichiers hiérarchique traditionnel
- Accès via URL : `s3://mon-bucket/backup/2026/janvier/fichier.zip`
- **Exemples** : AWS S3, Azure Blob Storage, Google Cloud Storage

### 3.7 Conteneurisation

#### Concepts Docker

|Concept|Description|
|---|---|
|**Image**|Template figé (ex: `nginx:latest`, `ubuntu:22.04`)|
|**Conteneur**|Instance en cours d'exécution d'une image|
|**Dockerfile**|Fichier de configuration pour créer une image|

#### Exemple de Dockerfile

```dockerfile
FROM python:3.11
COPY app.py /app/
RUN pip install flask
CMD ["python", "/app/app.py"]
```

#### Services Cloud de conteneurs

- **Azure Container Instances (ACI)** : Conteneurs sans gestion de serveur
- **AWS Fargate** : Exécution de conteneurs sans EC2
- **Google Cloud Run** : Déploiement d'API en conteneur en 1 clic

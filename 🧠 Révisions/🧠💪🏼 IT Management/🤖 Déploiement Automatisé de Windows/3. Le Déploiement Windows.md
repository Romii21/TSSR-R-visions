### 3.1 Processus standard

Le déploiement standard d'un Windows se déroule en 4 étapes :

```
1. Poste sans OS
      ↓ Boot (USB, réseau PXE…)
2. Chargement de WinPE (noyau léger)
      ↓
3. Assistant d'installation Windows
      ↓
4. Installation de Windows → Poste prêt
```

---

### 3.2 Le Master — Image de référence

Le **master** est une **image disque de référence** capturée à partir d'un ordinateur modèle. C'est cette image qui sera ensuite déployée sur les autres postes.

**Les 3 types de master :**

|Type|Contenu|Fréquence de mise à jour|Usage idéal|
|---|---|:-:|---|
|**Thin image**|OS uniquement|Tous les 1-2 ans|Les logiciels sont déployés après (GPO, MDT, scripts)|
|**Thick image**|OS + **tous** les logiciels|Maximum 6 mois|Environnement stable, mêmes logiciels partout|
|**Hybrid image**|OS + logiciels de base (antivirus, navigateur, bureautique)|Intermédiaire|Bon équilibre entre légèreté et praticité|

### 3.3 Sysprep — Préparer l'image

**Sysprep** _(System Preparation Tool)_ est l'outil qui **généralise** une image Windows pour qu'elle puisse être déployée sur d'autres machines sans conflit d'identifiants (SID, nom de machine…).

**Commande à exécuter sur le poste de référence :**

```cmd
C:\Windows\System32\sysprep\sysprep.exe /oobe /generalize /shutdown
```

|Option|Signification|
|---|---|
|`/oobe`|_Out-Of-Box Experience_ — lance la séquence de personnalisation au premier démarrage|
|`/generalize`|Supprime les infos spécifiques au matériel → déployable sur d'autres configs|
|`/shutdown`|Éteint le PC après le Sysprep|

> ⚠️ **IMPORTANT** : Après un Sysprep, **ne jamais rallumer le PC** avant d'avoir capturé l'image. Sinon le Sysprep est annulé.

### 3.4 WinPE — L'environnement de préinstallation

**WinPE** _(Windows Preinstallation Environment)_ est un **mini-OS Windows** utilisé pour démarrer un poste sans OS installé, afin de préparer et installer Windows.

- Existe depuis Windows XP
- Peut être lancé depuis une **clé USB, un CD, ou le réseau (PXE)**
- Remplace l'ancien MS-DOS avec beaucoup plus de fonctionnalités
- Compatible avec **MDT et WDS**

**Commandes utiles dans WinPE (via `diskpart`) :**

```cmd
diskpart              # Lancer l'utilitaire de gestion de disques
list disk             # Lister les disques disponibles
list volume           # Lister les volumes (partitions)
clean                 # Supprimer toutes les partitions du disque sélectionné
create partition primary  # Créer une partition principale
active                # Activer la partition (pour booter dessus)
create volume simple  # Créer un volume simple
```

### 3.5 PXE — Démarrage réseau

**PXE** _(Preboot Execution Environment)_ permet à un poste de **démarrer depuis le réseau** pour récupérer et installer un OS, sans support physique (USB/CD).

> 💡 PXE n'est pas spécifique à Windows. Par exemple, le projet **Syslinux** utilise PXE pour démarrer des OS Linux.

**Les 3 étapes du boot PXE :**

```
Étape 1 — DHCP/BOOTP
  Le client demande une adresse IP + l'emplacement du fichier d'amorçage
      ↓
Étape 2 — TFTP
  Le client télécharge le fichier d'amorçage depuis le serveur TFTP
      ↓
Étape 3 — Exécution
  Le fichier d'amorçage est exécuté (ex: PXELinux, WinPE)
```

**Détail des échanges réseau lors d'un boot PXE :**

|Étape|Protocole/Port|Description|
|---|---|---|
|DHCP-DISCOVER|UDP port 67|Le client demande une IP + l'option PXE|
|DHCP-OFFER|UDP port 68|Le serveur répond avec une IP + options PXE (60, 66, 67)|
|DHCP-REQUEST|UDP|Le client confirme l'adresse IP|
|DHCP-ACK|UDP|Le serveur valide → début du bail IP|
|NBP (Network Bootstrap Protocol)|UDP|Demande de téléchargement du fichier d'amorçage|
|Transfert fichier|TFTP|Envoi du fichier d'amorçage|
|Requête installateur|HTTP|Téléchargement de l'installateur Windows|

### 3.6 WIM — Format d'image disque

**WIM** _(Windows Imaging Format)_ est le format de fichier image disque utilisé par Microsoft pour le déploiement Windows.

- Extension : `.wim` ou `.swm`
- Existe depuis Windows XP SP2
- **Indépendant du matériel** (contrairement à Ghost ou .cue qui sont orientés secteurs)
- **Un seul fichier WIM peut contenir plusieurs images** (ex: Windows 10 Pro, Édition, Enterprise)
- Compatible avec **WADK, MDT et WinPE**

**Structure d'un fichier WIM :**

```
[ En-tête WIM ]
  ├── Ressources de fichiers (blocs compressés)
  ├── Métadonnées des ressources (descripteurs de dossiers, sécurité…)
  ├── Table d'adresses
  ├── Données XML (caractéristiques et infos sur l'image)
  └── Table d'intégrité
```

**Fichiers WIM importants dans une ISO Windows :**

|Fichier|Emplacement|Contenu|
|---|---|---|
|`install.wim`|`Sources\`|L'image d'installation de Windows|
|`boot.wim`|`Sources\`|L'environnement WinPE de démarrage|

### 3.7 DISM — Gestion des images WIM

**DISM** _(Deployment Image Servicing and Management)_ est l'outil de référence pour manipuler les images WIM. Il remplace l'ancien outil **ImageX**.

**Actions disponibles sur un fichier WIM :**

|Action|Description|
|---|---|
|**Mount / Unmount**|Monter/démonter l'image pour l'éditer|
|**Capture**|Copier une structure de fichiers vers un nouveau WIM|
|**Append**|Ajouter une image dans un WIM existant|
|**Export**|Extraire une image d'un WIM vers un nouveau WIM|
|**Delete**|Supprimer une image dans un WIM|
|**Info**|Afficher les métadonnées XML de l'image|

**Commandes DISM essentielles :**

Monter une image WIM pour la modifier :

```cmd
DISM /Mount-Wim /WimFile:Sources\fichier.WIM /index:<N°Image> /MountDir:<DossierDestination>
```

Enregistrer les modifications et démonter :

```cmd
DISM /Unmount-Wim /MountDir:<DossierDestination> /commit
```

Démonter sans enregistrer les modifications :

```cmd
DISM /Unmount-Wim /MountDir:<DossierDestination> /discard
```

Afficher les informations d'un WIM :

```cmd
DISM /Get-WimInfo /WimFile:Sources\fichier.WIM
```

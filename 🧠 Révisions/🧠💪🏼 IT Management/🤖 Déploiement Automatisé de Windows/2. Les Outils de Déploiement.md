### 2.1 WDS — Windows Deployment Services

**WDS** _(Services de déploiement Windows)_ est un rôle serveur Microsoft permettant d'installer Windows **via le réseau**.

- Successeur de **RIS** _(Remote Installation Service)_
- Supporte Windows Vista → 10 et Windows Server 2008 → 2016
- Disponible comme rôle sur Windows Server (depuis 2008 SP2)
- Une **partition dédiée** est nécessaire sur le serveur

**Ce que WDS fait :**

|Fonctionnalité|Description|
|---|---|
|Déploiement d'images **WIM**|Envoie l'image Windows aux clients|
|Boot **PXE** via **TFTP**|Fournit une image de démarrage par le réseau|
|Fichier de réponse XML|Automatise les réponses à l'assistant d'installation|

> 💡 WDS seul est suffisant pour un déploiement simple, mais il est **moins personnalisable** que MDT.

---

### 2.2 MDT — Microsoft Deployment Toolkit

**MDT** est une solution **gratuite** de Microsoft qui automatise la fabrication et l'installation d'images Windows à l'aide de **séquences de tâches**.

> ⚠️ MDT doit être utilisé **avec WADK** pour fonctionner.

**Deux modes de fonctionnement :**

|Mode|Description|Intervention humaine|
|---|---|:-:|
|**LiteTouch**|MDT seul|✅ Oui (minimale)|
|**ZeroTouch**|MDT + SCCM|❌ Non (totalement automatique)|

> ⚠️ **Limitation importante** : MDT n'installe pas de service TFTP → pas de démarrage PXE natif.  
> **Solution** : Utiliser **MDT + WDS** ensemble.

**Fonctionnalités de personnalisation MDT :**

|Fonctionnalité|Détail|
|---|---|
|Nom de l'ordinateur|Nommage automatique des machines|
|Jonction au domaine|Intégration automatique à l'AD|
|Installation d'applications|Silencieuse ou via séquence de tâches|
|Exécution de scripts|VBScript, CMD, **PowerShell**|
|Sauvegarde/Restauration des profils|Migration utilisateur|
|Activation de **BitLocker**|Chiffrement du disque|
|Installation des pilotes|Drivers intégrés à l'image|

---

### 2.3 WADK — Windows Assessment and Deployment Kit

**WADK** _(anciennement WAIK)_ est une suite d'outils Microsoft pour le déploiement d'OS. **Nécessaire au fonctionnement de MDT.**

|Outil|Nom complet|Rôle|
|---|---|---|
|**USMT**|User State Migration Tool|Migration des données utilisateur|
|**ACT**|Application Compatibility Toolkit|Tester la compatibilité des apps après migration|
|**DISM**|Deployment Image Servicing and Management|Gérer les images WIM (pilotes, mises à jour…)|
|**WinPE**|Windows Preinstallation Environment|Environnement léger pour le déploiement|
|**VAMT**|Volume Activation Management Tool|Activation de licences via clé MAK ou KMS|
|**Sysprep**|System Preparation Tool|Préparer/généraliser une image pour capture|

### 2.4 SCCM — System Center Configuration Manager

**SCCM** est un logiciel Microsoft de gestion de parc informatique à grande échelle. C'est une solution **payante** (licence propriétaire).

> 💡 SCCM est surtout utile pour les **très grands parcs informatiques**. Il intègre WDS et MDT.

**Fonctionnalités SCCM :**

|Fonctionnalité|Description|
|---|---|
|Déploiement OS via image **WIM**|Déploiement complet automatisé (**ZeroTouch**)|
|Prise de main à distance|Administration des postes clients|
|Gestion de correctifs|Déploiement des mises à jour|
|Télédistribution d'applications|Installation à distance de logiciels|
|Inventaire matériel et logiciel|Connaissance du parc|
|Gestion de la conformité|Vérification des configurations|
|Administration des politiques de sécurité|GPO et sécurité centralisées|

**Architecture SCCM :**

```
Site Central d'Administration (CAS)
  └── Gère jusqu'à 25 sites primaires
      Site Primaire → jusqu'à 100 000 clients
        └── Sites Secondaires → jusqu'à 5 000 clients
              └── Points de distribution (DP) → jusqu'à 4 000 clients
                    └── Ordinateurs clients
```

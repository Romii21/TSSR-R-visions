### 2.1 Les terminaux IP

Un **terminal IP** est tout appareil fonctionnant sur un réseau IP et permettant de passer des appels.

|Type de terminal|Description|
|---|---|
|Téléphone IP|Téléphone classique connecté en RJ45, fonctionne directement sur IP|
|Smartphone|Terminal mobile avec softphone ou appli VoIP|
|Ordinateur|Via un logiciel softphone|

---

### 2.2 Les switchs PoE

**PoE** = _Power over Ethernet_ — Standard permettant d'**alimenter électriquement un équipement via le câble RJ45**, sans prise secteur.

|Norme|Puissance max.|Usage typique|
|---|---|---|
|**IEEE 802.3af**|15,4 W|Téléphones IP, caméras IP|
|**IEEE 802.3at** (PoE+)|30 W|Points d'accès Wi-Fi|
|**IEEE 802.3bt** (PoE++)|60-100 W|Équipements plus puissants|

> 💡 **Avantage majeur :** un **seul câble RJ45** pour les données ET l'alimentation. Plus besoin de prise électrique près de chaque téléphone.

---

### 2.3 L'adaptateur ATA

**ATA** = _Adaptateur Téléphone Analogique_ (ou Adaptateur Terminal Analogique)

C'est un **convertisseur analogique ↔ IP** permettant de connecter un vieux téléphone analogique (réseau RTC) à un réseau IP.

```
[Téléphone analogique] --RJ11/USB--> [ATA] --RJ45--> [Réseau IP]
```

> 🔌 **Ports :** côté analogique = RJ11 (ou USB) | côté IP = RJ45

---

### 2.4 Les softphones

Un **softphone** est un **logiciel remplaçant le téléphone physique**. On parle aussi de **softphonie** ou de **logiciel SIP**.

|Caractéristique|Détail|
|---|---|
|Installation|Ordinateur de bureau, portable, smartphone, tablette|
|Protocole utilisé|SIP (principalement)|
|Fonctionnalités|Appels, vidéo, messagerie instantanée, transfert de fichiers|
|Exemples|Linphone, Zoiper, MicroSIP, Teams (Teams Phone)|

---

### 2.5 Le commutateur (PBX)

Un **commutateur** (ou **autocommutateur**, **autocom**, **PBX**) est l'équivalent d'un **central téléphonique privé** : il redirige les flux d'appels en interne et vers l'extérieur.

**Deux types principaux :**

| |PABX|IPBX|
|---|---|---|
|Réseau|RTC / RNIS|IP (LAN / Internet)|
|Câblage|Câblage cuivre dédié|Câble RJ45 standard|
|RNIS|Oui (via cartes additionnelles)|Oui|
|Protocole|Propriétaire|SIP / H.323|

> 📌 La solution **Centrex** peut être vue comme un autocommutateur **externalisé dans le cloud**.

---

### 2.6 Le PABX

**PABX** = _Private Automatic Branch EXchange_

Autocommutateur téléphonique privé fonctionnant sur **réseau RTC**.

```
Architecture avec PABX :
[Tél. analogique] ──┐
[Tél. IP]          ──┤→ [PABX] → [Réseau RTC/RNIS]
[Passerelle VoIP]  ──┘
```

- Gère les appels sur le **réseau interne** de l'entreprise
- Relie le réseau interne au **réseau téléphonique public**
- Peut fonctionner sur **RNIS** avec des cartes additionnelles

---

### 2.7 L'IPBX

**IPBX** = _Internet Protocol Branch EXchange_ (aussi appelé PABX IP)

Autocommutateur utilisant le **protocole IP**, pouvant communiquer sur Internet.

```
Architecture avec IPBX :
[Téléphone IP] ──┐
[Smartphone]   ──┤→ [IPBX] → [Internet]
[PC portable]  ──┤
[Softphone]    ──┘
```

- Même rôle que le PABX, mais sur **réseau IP**
- Les transmissions passent par le **LAN** (câble RJ45 standard)
- Peut aussi gérer du **RNIS**
- Supporte la **softphonie**, les téléphones IP et les smartphones

> 📌 Différence clé : PABX → câblage cuivre dédié | IPBX → réseau IP standard

---

### 2.8 La passerelle VoIP

**Passerelle VoIP** _(VoIP Gateway)_ : dispositif qui **convertit le trafic téléphonique** en données transmissibles sur un réseau informatique.

> ⚠️ À ne pas confondre avec une **passerelle multimédia** _(media gateway)_ !

**Rôle :** faire le pont entre le réseau RTC (analogique/numérique) et le réseau IP.

**Ports typiques :**

- **FXS** _(Foreign eXchange Subscriber)_ : connecte les téléphones analogiques
- **FXO** _(Foreign eXchange Office)_ : connecte les lignes téléphoniques RTC

---

### 2.9 Le protocole H.323

|Caractéristique|Détail|
|---|---|
|Origine|Dérivé du H.320 (utilisé sur RNIS)|
|Défini par|UIT-T _(Union Internationale des Télécommunications)_|
|1ère version|1996|
|Dernière version|2009|
|Statut|**Remplacé par SIP**|
|Description|Système de communication multimédia (voix, image, données) en mode paquet|

### 1.1 Qu'est-ce qu'Active Directory ?

**Active Directory (AD)** est l'implémentation par Microsoft des services d'annuaire basés sur le protocole **LDAP (Lightweight Directory Access Protocol)** pour les systèmes Windows.

**Objectifs principaux :**

- Centraliser l'identification et l'authentification des utilisateurs
- Gérer les politiques de sécurité sur l'ensemble du réseau
- Répertorier et faciliter la gestion des ressources réseau

### 1.2 Historique

|Année|Événement|
|---|---|
|1996|Création sous le nom **NTDS (NT Directory Services)**|
|1999|Première utilisation majeure avec Windows 2000 Server|
|Evolution|Passage de la base **SAM (Security Account Manager)** à LDAP|
|Evolution|Changement du moteur de stockage de Jet à **ESENT**|

### 1.3 Différence avec SAM

**SAM (Security Account Manager)** était utilisé avant AD pour gérer les comptes localement. Limites de SAM :

- Adapté uniquement aux petits réseaux (jusqu'à ~20 machines)
- Aucune centralisation
- Gestion complexe au-delà de quelques dizaines de machines

### 1.4 Base de données hiérarchique vs relationnelle

|Base Hiérarchique (LDAP/AD)|Base Relationnelle (SGBD)|
|---|---|
|Structure en arborescence|Tables avec relations|
|Recherche par chemin hiérarchique|Recherche par clés et jointures|
|Optimisée pour la lecture|Optimisée pour lectures/écritures|
|Exemple : AD, OpenLDAP|Exemple : MySQL, Oracle|

### 1.5 Part de marché et capacités

**Chiffres clés :**

- 25% du marché officiel (potentiellement 75% en réalité)
- Capacité : plus de **2 milliards d'objets** créables
- Maximum de **999 GPO** applicables à un objet
- Maximum de **1015 appartenances** de groupe par objet

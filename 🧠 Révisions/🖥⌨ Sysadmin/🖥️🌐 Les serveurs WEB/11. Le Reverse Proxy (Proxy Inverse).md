### Définition

Un **reverse proxy** (proxy inverse) est un serveur placé **devant les serveurs backend** qui :

- Reçoit les connexions des clients
- Redirige le trafic vers les serveurs appropriés
- Renvoie les réponses aux clients

### Architecture

```
Internet ──> Reverse Proxy ──┬──> Serveur Backend 1
                             ├──> Serveur Backend 2
                             └──> Serveur Backend 3
```

### Fonctionnalités principales

|Fonctionnalité|Description|Bénéfice|
|---|---|---|
|**Cache**|Stocke les contenus statiques|Performances accrues|
|**Répartition de charge**|Distribue les requêtes|Scalabilité|
|**SSL/TLS Termination**|Gère le HTTPS|Simplification backend|
|**Sécurité**|Cache les serveurs réels|Protection|
|**Compression**|Compresse les réponses|Bande passante réduite|

### Solutions de Reverse Proxy

|Solution|Caractéristiques|
|---|---|
|**Nginx**|Rapide, léger, très utilisé|
|**Apache**|Module mod_proxy, flexible|
|**HAProxy**|Spécialisé répartition de charge|
|**Varnish**|Cache HTTP ultra-performant|
|**Traefik**|Moderne, orienté conteneurs|

### Configuration Nginx en Reverse Proxy

```nginx
# /etc/nginx/sites-available/reverse-proxy
server {
    listen 80;
    server_name example.com;
    
    location / {
        # Transférer vers le backend
        proxy_pass http://localhost:8080;
        
        # Passer les informations du client
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Configuration Apache en Reverse Proxy

```apache
# Activer les modules nécessaires
sudo a2enmod proxy proxy_http

# /etc/apache2/sites-available/reverse-proxy.conf
<VirtualHost *:80>
    ServerName example.com
    
    ProxyPreserveHost On
    ProxyPass / http://localhost:8080/
    ProxyPassReverse / http://localhost:8080/
</VirtualHost>
```

### Cas d'usage typique : Nginx devant Apache

**Nginx** excelle pour le contenu statique, **Apache** pour le contenu dynamique.

```
Client ──> Nginx (port 80)
           │
           ├──> Fichiers statiques (images, CSS, JS)
           │
           └──> Apache (port 8080) ──> PHP/Backend
```

#### Configuration combinée

```nginx
server {
    listen 80;
    server_name example.com;
    
    # Contenu statique servi directement par Nginx
    location ~* \.(jpg|jpeg|png|gif|ico|css|js|svg|pdf)$ {
        root /var/www/html;
        expires 30d;
    }
    
    # Contenu dynamique transmis à Apache
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

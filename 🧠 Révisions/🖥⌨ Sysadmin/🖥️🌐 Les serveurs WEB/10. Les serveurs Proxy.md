### Définition

Un **serveur proxy** (ou serveur mandataire) est un **intermédiaire** entre le client et le serveur final.

### Caractéristiques

|Aspect|Détail|
|---|---|
|**Couche**|Couche 7 (Application) du modèle OSI|
|**Types**|Spécifiques (HTTP, FTP) ou génériques (SOCKS)|
|**Position**|Entre client et serveur|

### Types de proxy

|Type|Description|Usage|
|---|---|---|
|**Proxy direct**|Le client utilise le proxy volontairement|Anonymat, filtrage|
|**Proxy transparent**|Le client ne le sait pas|Portail captif, filtrage réseau|
|**Proxy inverse**|Mandataire côté serveur|Répartition de charge, cache|

### Fonctionnalités du Proxy HTTP

|Fonctionnalité|Description|
|---|---|
|**Cache**|Stocke les contenus fréquents en mémoire|
|**Filtrage**|Bloque certains contenus ou sites|
|**Journalisation**|Enregistre les requêtes (monitoring)|
|**Modification**|Traduction, compression à la volée|
|**Anonymat**|Masque l'adresse IP du client|

### Problème avec HTTPS

- HTTPS chiffre les données de bout en bout
- Pour inspecter le trafic, le proxy doit **casser le tunnel TLS**
- Cela constitue une **attaque de l'intercepteur (MitM)**
- Nécessite l'installation d'un certificat racine sur les clients

### Exemple : Squid

**Squid** est un proxy HTTP populaire et open-source.

```bash
# Installation
sudo apt-get install squid

# Configuration de base
# /etc/squid/squid.conf
http_port 3128
cache_dir ufs /var/spool/squid 100 16 256
access_log /var/log/squid/access.log

# Démarrer Squid
sudo systemctl start squid
```

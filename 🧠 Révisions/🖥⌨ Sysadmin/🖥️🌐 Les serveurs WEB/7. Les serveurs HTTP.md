### Rôle d'un serveur web

Un **serveur HTTP** est un logiciel qui :

- Écoute sur les ports **80** (HTTP) et **443** (HTTPS)
- Reçoit les requêtes HTTP des clients
- Fournit les réponses HTTP correspondantes

### Fonctionnement classique

```
Client ──[Requête HTTP]──> Serveur Web
                           │
                           ├─ Trouve le fichier
                           │  dans l'arborescence
                           │
Client <──[Réponse HTTP]── Retourne la ressource
```

### Principaux serveurs HTTP

|Serveur|Caractéristiques|Part de marché|
|---|---|---|
|**Apache**|Très répandu, modulaire, flexible|~30%|
|**Nginx**|Performant, léger, reverse proxy|~34%|
|**Microsoft IIS**|Pour Windows Server|~6%|
|**LiteSpeed**|Rapide, compatible Apache|~10%|
|**Autres**|Caddy, Tomcat, Node.js, etc.|~20%|

### Installation et commandes de base

#### Apache

```bash
# Installation (Ubuntu/Debian)
sudo apt-get install apache2

# Démarrer Apache
sudo systemctl start apache2

# Arrêter Apache
sudo systemctl stop apache2

# Redémarrer Apache
sudo systemctl restart apache2

# Vérifier le statut
sudo systemctl status apache2

# Tester la configuration
sudo apache2ctl configtest
```

#### Nginx

```bash
# Installation (Ubuntu/Debian)
sudo apt-get install nginx

# Démarrer Nginx
sudo systemctl start nginx

# Arrêter Nginx
sudo systemctl stop nginx

# Redémarrer Nginx
sudo systemctl restart nginx

# Recharger la configuration
sudo nginx -s reload

# Tester la configuration
sudo nginx -t
```

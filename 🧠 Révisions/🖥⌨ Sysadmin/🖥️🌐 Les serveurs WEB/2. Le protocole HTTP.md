### Caractéristiques principales

**HTTP** (HyperText Transfer Protocol) est un protocole **client-serveur** qui permet l'échange de données sur le Web.

|Aspect|Détails|
|---|---|
|**Versions**|HTTP/1.1, HTTP/2, HTTP/3 (rare, basé sur QUIC)|
|**Transport**|Basé sur TCP (sauf HTTP/3)|
|**Port standard**|80|
|**Fonctionnement**|Requête ➜ Réponse|
|**État**|Sans état (stateless) - aucune mémoire entre requêtes|

### Structure des messages HTTP

Les messages HTTP (requêtes et réponses) sont des **messages textes** composés de :

- **Une ligne de démarrage** : définit le type de message
- **Des entêtes** (headers) : métadonnées sur la communication
- **Un corps** (body) : données transmises (optionnel)

> **Note** : Bien que HTTP soit sans état, les cookies HTTP permettent de créer des sessions avec état pour maintenir un contexte entre les requêtes.

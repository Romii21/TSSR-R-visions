### Principe

Un seul serveur web peut héberger **plusieurs sites** distincts grâce au mécanisme de **Virtual Hosts** (Apache) ou **Server Blocks** (Nginx).

### Critères de différenciation

|Critère|Description|Exemple|
|---|---|---|
|**Nom d'hôte**|Le plus courant|`site1.com` vs `site2.com`|
|**Adresse IP**|Plusieurs IPs sur le serveur|`192.168.1.10` vs `192.168.1.11`|
|**Port**|Même IP, ports différents|`:80` vs `:8080`|

### Configuration Apache (Virtual Host)

```apache
# /etc/apache2/sites-available/site1.conf
<VirtualHost *:80>
    ServerName site1.example.com
    DocumentRoot /var/www/site1
    ErrorLog ${APACHE_LOG_DIR}/site1-error.log
</VirtualHost>

# Activer le site
sudo a2ensite site1.conf
sudo systemctl reload apache2
```

### Configuration Nginx (Server Block)

```nginx
# /etc/nginx/sites-available/site1
server {
    listen 80;
    server_name site1.example.com;
    root /var/www/site1;
    
    location / {
        try_files $uri $uri/ =404;
    }
}

# Activer le site
sudo ln -s /etc/nginx/sites-available/site1 /etc/nginx/sites-enabled/
sudo systemctl reload nginx
```

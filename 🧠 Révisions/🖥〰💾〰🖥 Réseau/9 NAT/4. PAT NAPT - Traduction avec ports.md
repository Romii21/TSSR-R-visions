### Définition

**NAPT (Network Address and Port Translation)** ou **PAT (Port Address Translation)** :

- Traduction **dynamique** de plusieurs flux internes vers **une seule IP publique**
- Les **ports** servent à différencier les communications
- Défini dans la **RFC 2663**

**Autres noms :**

- NAT overload
- NAT masquerade (Linux)
- SNAT avec ports

### Caractéristiques

- **Sens** : Source
- **Mode** : Dynamique
- **Niveau** : @IP + port

### Fonctionnement

```
[Machine interne]           [Routeur NAT]              [Internet]
10.1.1.11:52369    →    203.1.113.123:52369    →    216.58.214.83:443
                         (IP publique)
```

**Étape 1 - Requête sortante :**

1. Machine `10.1.1.11` veut accéder à `216.58.214.83:443` (HTTPS)
2. Le routeur enregistre dans sa table : `10.1.1.11:52369 ↔ 203.1.113.123:52369`
3. Le routeur **remplace l'adresse source** et envoie sur Internet

**Étape 2 - Réponse entrante :**

1. Le serveur répond à `203.1.113.123:52369`
2. Le routeur cherche dans sa table le port `52369` → trouve `10.1.1.11`
3. Le routeur **transmet le paquet** vers la machine interne

### Gestion des collisions de ports

Si le port est déjà utilisé par une autre machine interne : ➡️ Le routeur choisit un **autre port disponible** (ex : 52370)

### Usage

✅ Le NAT **le plus utilisé** à cause de la pénurie d'IPv4  
✅ Fournir un accès Internet aux machines clientes

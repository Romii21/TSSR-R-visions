### Définition

**DNAT (Destination Network Address Translation)** :

- Traduction **statique** de l'adresse de destination
- Redirige une communication entrante vers une **machine interne**
- Aussi appelé **port forwarding**

### Caractéristiques

- **Sens** : Destination
- **Mode** : Statique
- **Niveau** : @IP (+ éventuellement port)

### Fonctionnement

```
[Internet]              [Routeur NAT]              [Serveur interne]
204.1.97.10:57221  →  203.1.113.123:80  →  172.16.1.15:80
                       (IP publique)
```

**Étape 1 - Requête entrante :**

1. Client externe demande `203.1.113.123:80`
2. Le routeur traduit : `203.1.113.123:80` → `172.16.1.15:80`
3. Le paquet est transmis au serveur interne

**Étape 2 - Réponse sortante :**

1. Le serveur répond : `172.16.1.15:80` → `204.1.97.10:57221`
2. Le routeur remplace l'adresse source : `172.16.1.15` → `203.1.113.123`
3. Le paquet est envoyé vers Internet

### Multiplexage de ports

Plusieurs services peuvent être publiés sur **une même IP publique** avec des ports différents :

```
203.1.113.123:80    →  172.16.1.15:80   (HTTP)
203.1.113.123:3389  →  172.16.1.15:3389 (RDP)
```

### Usage

✅ Publication de services internes (HTTP, HTTPS, RDP, FTP, etc.)
